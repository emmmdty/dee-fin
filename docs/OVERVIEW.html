把一堆新闻，变成一张信得过的事件图

机器读文章、抽事件、连因果、建成图谱，这件事早有人做。 难的是——建出来的图有错，你敢不敢拿它去做决策？ 这个课题给「敢不敢」一个可量化的答案，并且在下游任务的验收下把图修好。

这个课题到底在做什么

一句话：让机器自动读文本、建一张事件之间的因果图，并且对这张图的每一处错误负责到底。

先看「事件图谱」长什么样

随便拿一段新闻：

示例文档（贯穿全文） 周二凌晨发生 6.2 级地震。震中多栋建筑倒塌， 造成至少 30 人受伤。当地政府随即启动救援行动。

人一眼就懂：地震 → 倒塌 → 受伤 → 救援。机器要做的是把这个理解写成结构化数据：

  • 节点＝一个事件。带类型（Catastrophe）、触发词（「地震」）、论元（时间／地点／伤亡人数）、以及它在原文的哪个字符位置。
  • 边＝事件之间的关系。因果（CAUSE / PRECONDITION）、子事件（subevent）、时序（BEFORE）。
  • 整张图＝事件图谱（Event Knowledge Graph）。

为什么值得写一篇学位论文

因为上面每一步机器都会做错，而且错误会往下游滚雪球：

  • 同一件事在文中被提了三次（「地震」「这场灾害」「6.2 级强震」）——机器可能建成 3 个节点，也可能把两件不同的事错误地合并成 1 个。→ 身份错
  • 因果关系极其稀疏且难判。本项目当前的抽取器在测试集上只召回了 0.4% 的因果边（810 条里抓到 3 条）。→ 结构错
  • 文中说的「预计将追加救援」是还没发生的事，机器却当成既成事实建进图里。→ 事实错
  • 上面三种错叠加，导致下游预测「接下来会发生什么」时，答案根本不在图里能到达的范围内。→ 传播失效

市面上绝大多数工作只优化「抽得准不准」这一个指标。本课题问的是另一个问题：

我建的图有错，那么我用它做出的下游判断，错的概率上界是多少？ 以及——能不能让下游任务来当裁判，反过来把图修对？

论文的骨架就是把「可信」拆成四层，每层一章，串成一条流水线。前三章负责把图建可信，第四章负责把误差算清楚并闭环修复——第四章是全篇的招牌（headline）。

02 / 词汇表

名词扫盲

这一节把后面会反复出现的词一次讲清楚。只看这一节就能读懂全文。

事件图谱 Event Knowledge Graph
节点是事件（地震、倒塌）而不是实体（苹果公司、张三）的知识图谱。边表示事件之间的因果、时序、包含关系。
MAVEN 四件套 THU-KEG
清华开放的中文/英文事件标注数据集家族，同一批 4480 篇文档上标了四层：MAVEN（事件检测）、MAVEN-Arg（论元）、MAVEN-ERE（关系＋共指）、MAVEN-FACT（事实性）。四章各用一层，天然对齐——这是选它的核心原因。官方 test 标签不公开，本地一律用 valid 报数。
CGEP Causality Graph Event Prediction
下游任务：给你一张已建好的事件因果图，图里有个位置空着（比如「救援」），从 512 个候选事件里挑出正确的那个。这就是 Ch4 用来验收图质量的裁判。
ECG Event Causality Graph
CGEP 的一个题目单位：一篇文档内由因果＋子事件边连成的连通子图，要求至少 4 个节点。本项目从 MAVEN-ERE 重建了 2994 个，平均 8.82 节点 / 13.21 边。
SeDGPL 基座模型
2024 年的 CGEP 最强公开方法，本项目自己复现并当基线。三个部件：DsGL 把图压成一串文字喂给语言模型、EeCE 两级门控编码、ScEP 对比学习的预测头。注意：它不用论元，只吃（事件类型，触发词，所在句子）。
MRR / Hits@k 排序指标
候选排序题的标准分。MRR＝正确答案排名倒数的平均值，越高越好。本项目自跑 SeDGPL 基线 MRR ＝ 0.1836。
Conformal Prediction 保形预测
一套不依赖模型假设的统计工具：不再只给一个答案，而是给一个候选集合，并数学保证「正确答案落在集合里」的概率 ≥ 1−α。α 是你自己设的风险预算（比如 0.1 就是允许错 10%）。集合越小越有价值。
可达性 reachability
正确答案在图上能不能沿着边走到。如果 Ch2 把一条关键的因果边删了，答案就永远到不了——这种错误是下游模型自己察觉不到的，必须单独给它留风险预算。这是本课题最硬的差异点。
CRC Conformal Risk Control
保形预测的推广：可以直接控制「漏检率（FNR）不超过 α」这类风险。Ch2 用它决定哪些边准入——宁可多留几条噪声边，也要保证不漏掉关键边。
校准 / ECE calibration
模型说「我 90% 有把握」时，是不是真有 90% 对。ECE 衡量这个偏差。Ch1 输出的 node_confidence 必须是校准过的，Ch4 的误差预算才吃得下。
Repair trace 修复轨迹
结构化记录「我改了哪条边、因为触发了哪条一致性规则、改前改后差多少」。这是「可追溯」这个卖点的落地形式。
Registry 插件注册表
本项目的工程约定：每个可替换组件用 @register("名字") 注册，配置文件按名字选。升级一个阶段 = 加一个实现，不返工调用方。

Ch1 · 身份可信Phase C / C2 · 尚未开工

同一件事，只建一个节点

想解决什么问题

一篇文档里同一个事件会被反复提及，不同文档之间更是如此。如果不做归并，图里就会出现一堆重复节点； 归并太狠，又会把两件相似但不同的事错误合并（比如同一天的两次余震）。

难点：相似事件难例 「震中多栋建筑倒塌」和「三小时后一栋危楼再次垮塌」—— 事件类型相同、触发词近义、同一篇文档。该合并吗？不该。 普通共指模型在这种「难例」上误合并率很高。

本章的差异化不在共指本身（那是复现），而在两点：难例判别，和输出一个 下游能消费的校准置信 node_confidence——它直接喂给 Ch4 的误差预算。

Ch2 · 结构可信Phase A / B · 全篇关键路径

把边连对，连错了留下账

想解决什么问题

节点有了，事件之间到底谁导致谁？这是整个课题最难、也是目前最卡的一环。三个具体麻烦：

  • 极度稀疏：一篇文档 n 个事件就有 n² 个候选对，真正有因果关系的只有零星几对。不处理类不平衡，模型会学会「全部预测无关系」。
  • 局部矛盾：逐对独立分类会产生 A 在 B 前、B 在 C 前、C 又在 A 前这种环，或者违反传递性的时序。
  • 删错代价不对称：多留一条噪声边只是变吵；删掉一条关键因果边，下游答案就永远到不了。所以准入不能只看准确率。

Ch3 · 事实可信Phase D · 尚未开工

图里那些「其实没发生」的事件

想解决什么问题

抽取器只管「文中提到了一个事件」，不管这个事件到底发生了没有。新闻里大量事件是否定的、假设的、传闻的、未来的。 把它们当成既成事实建进图，下游推理就是在错误前提上做的。

MAVEN-FACT 的五类事实性 CT+ 确定发生（「建筑倒塌了」）　· CT− 确定没发生（「大坝并未溃决」）　· PS+ 可能发生（「或将追加救援」）　· PS− 可能没发生　· Uu 无法判断

本章的创新点位置需要非常小心：「用结构信息帮助判断事实性」这件事 MAVEN-FACT 原文已经证过了，不能主张为新。 本章的 delta 是另外两件事：

  1. 预测图上的鲁棒性——MAVEN-FACT 原文全部用 gold 输入（gold 论元、gold 关系）。真实系统吃的是 Ch1/Ch2 输出的有错的图。掉多少分？没人量化过。
  2. 事实性驱动的图净化——把非事实事件剔除或降权后，图变干净了，下游预测有没有变好？这个闭环没人做过。

Ch4 · 传播可信 ★ headlinePhase E / F · 半成品最多的一章

让下游当裁判，反过来把图修对

想解决什么问题

前三章各自把自己那层做好了，但没人回答：这些错误合起来，对最终决策的伤害有多大？ 而且，现在流行的「自我修复」（self-refine）有个已知毛病——模型自己觉得改好了，实际分数掉了。

Ch4 用一个具体的下游任务（CGEP：预测下一个事件）同时解决这两件事：

  1. 误差传播：对比 gold / predicted / repaired 三张图，量化哪一类错误最伤下游。
  2. 下游门控闭环修复：控制器提出一个最小代价编辑（加边、删边、并节点……），只有当 SeDGPL 的预测成绩真的变好时才接受这次编辑。这就是治 self-refine 掉点的办法。
闭环长什么样 观测（Ch2 一致性违规 + Ch3 事实性 + 证据）→ 控制器提议「补一条 受伤 → 救援 的 PRECONDITION 边」→ 风险闸门检查 → 重跑 SeDGPL，MRR 涨了 → 接受；没涨 → 回滚 → 迭代到收敛 → 输出 repair trace。

04 / 全篇统一

误差预算：把「敢不敢信」算成一个数

这是把四章串成一篇论文而不是四篇小文章的那根线。

核心洞察是一个别人容易漏掉的点：上游删边造成的错误，下游模型察觉不到。

Ch2 的准入如果把「受伤 → 救援」这条边删了，那么「救援」这个答案在图上根本走不到。 推理阶段的校准器只能看见「我排序排错了」，看不见「正确答案压根不在候选可达范围内」。 所以必须为这种不可达单独留一份风险预算——这就是 reachability 预算，本课题最硬的差异点。

07 / 如实

诚实区：已知的负结果与止损线

这一节存在的意义是防止未来的自己（或者 agent）自欺。下面每一条都是实测，不是预期。

已经撞上的两堵墙

墙一 · M1 / M2 都是噪声级

按 BFS 距离选边（M1）和加 reach_anchor 结构 bit（M2）， GPU A/B 实测 ΔMRR 分别是 +0.005 和 −0.0015 —— 都在噪声范围内。 两者如实进消融附录，不当卖点。M2 的结论是明确的负结果。

墙二 · 抽取器因果召回 0.4%

生成式 SFT+GRPO 在 MAVEN-ERE 上 causal 3/810、subevent 0/139。 构建版 ECG 直接退化到不可扫描。所以 M3b 只能落成受控扫描而非真实图闭环。 这个 0.4% 本身作为诚实数据点写进论文。

数据口径上的坑（写作时会被审稿人抓）

  • ESC 文档级切分会泄漏。同一 topic 就是同一个故事。实测 topic-CV 0.0599 vs doc-split 0.1802 ≈ 复现的 0.196 —— SeDGPL 论文公开的 19.6 依赖的正是这个泄漏切分。协议一律用 topic 交叉验证。
  • SeDGPL 的 MAVEN 版数据没发布，论文里的 27.9 不可比。主表必须用我们自跑的基线，引用公开数字要显式标注「原论文数据构建，非同数据可比」。
  • MAVEN 系列官方 test 标签隐藏，走 CodaLab。本地一律 train 调参、valid 报数，test 不进本地调参。
  • 词表 transductive 是有意为之：<a_i> token 清单覆盖 train+test，与 SeDGPL 的 to_add.json 一致——只有 token 清单跨切分，没有标签／图／梯度泄漏。这点要在论文里主动说明。
