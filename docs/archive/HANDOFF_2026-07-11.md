## 2. 关键一手发现（修正/推翻了 07-10 的前提）

**全部为本轮实测，勿再质疑。**

1. **SeDGPL 的 MAVEN 版数据未发布。** `load_data.py` 读 `data/MAVENSubWoRe.npy`，repo 中不存在；
   只发布了 `ESCSubWoRe.npy`（ESC 版）。→ 论文 CGEP-MAVEN **27.9 不是同数据可比数**。
2. **官方发布的数据喂不进官方发布的代码。** `ESCSubWoRe.npy` 按 EventStoryLine **topic** 组织
   （`data[topic][doc_id]=[实例]`，22 topic / 244 文档 / 1192 实例），而 `load_data.py` 读的是
   `data['train']`。ESC 必须按 topic 交叉验证（文档级切分会泄漏，见 §4）。
3. **CGEP 只消费 (事件类型, 触发词, 所在句子)，不用论元**（`tools.getTemplate`/`getSentence`）
   → MAVEN-ARG 不在闭环关键路径，**ch1 本轮只做事件检测，论元延后**。
4. **查询边判据是「尾节点出度 0 且入度 1」**（不是 07-10 设想的仅出度 0）。从 `getTemplate` 推导：
   每条模板边两端都渲染成 `<a_i>` token，gold 若出现在其他边就把答案印进 prompt。
   **在官方 ESC 的 1192 个实例上 1192/1192 成立**（一手验证，非猜想）。
5. **DsGL 截断是「按存储顺序取前 20 条」**，最短路距离只用于**排序**幸存边（远边在前）。
6. **候选采样（从 ESC 反推）**：256 个互不相同的事件**节点**（非触发词串），gold 必在内，
   全语料近均匀采样，**不排除本图节点**。触发词会碰撞 → ESC 有效答案 178.0/256、我们 MAVEN 356.5/512
   （比值 0.696≈0.693，吻合）。精确可抽样清单不可复原（ESC 有 161 候选不在任何 ECG）。

## 3. Phase 0：CGEP-MAVEN 重建（已验收）

口径（`succession/data/cgep.py` 权威）：ECG = 文档内 **causal(CAUSE+PRECONDITION)+subevent** 的
无向连通分量，**节点 ≥ 4**；时序 BEFORE **不进拓扑**（加它 → 235 边/ECG，20 边预算放不下），
只作 M2 结构特征。查询边 = 出度 0 且入度 1 的边。候选 512，同 split 均匀采样。

`uv run python scripts/build_cgep.py --split train+valid --report-stats`：

| 指标 | 论文 | 我们 | 判定 |
|---|---|---|---|
| 文档数 | 3015 | 2994 | PASS（±2%） |
| 节点/ECG | 8.4 | 8.82 | PASS（±0.5） |
| 边/ECG | 12.9 | 13.21 | PASS（±0.5） |
| ECG 数 | 5308 | 3743 | **已知偏差**（记录，不 tune） |
| 实例数 | 12200 | 10116 | 记录 |

ECG 数对不上是 `MAVENSubWoRe.npy` 未发布的直接后果，如实写出。**主表因此必须以我们自跑的 SeDGPL
为基线**，公开数字全标「原论文数据构建，非同数据可比」。

## 4. Phase 1：SeDGPL 复现锚（已验收）★核心发现

**验收 (a) 数据不变量**：加载器+DsGL 在官方 ESC 上复现 22 topic / 244 文档 / 1192 实例、候选恒 256、
query-edge 判据 1192/1192、有效答案 178.0/256、38.8% 实例被 20 边预算截断、文本与类型模板分词等长
（SeDGPL 自己的 assert）。全部通过。

**验收 (b) 模型**（均为五折均值，0 unscorable，1192 实例全计入）：

| 配置 | MRR | 说明 |
|---|---|---|
| **topic 交叉验证**（我们的协议） | **0.0599 ± 0.0138** | random 0.0286 / frequency 0.0217 |
| **document 随机切分**（论文口径） | **0.1802 ± 0.0089** | 复现论文 0.196；random 0.0288 / frequency 0.0491 |
| topic, lr 1e-6→**5e-6** | 0.0845→0.0701（fold0） | **学习率不是原因** |

**结论：SeDGPL 的 ESC 19.6 依赖同 topic 泄漏。** 隔离验证：① lr 调高反降，排除欠训练；
② random 两切分相同（候选难度一样）；③ 节点身份**从不泄漏**（0%），泄漏在**触发词层面**
（测试 gold 触发词曾是训练 gold：document 43.3% vs topic 29.7%）——同 topic 的文档讲同一个故事，
按文档切分让模型学到 topic 特异的因果链。④ 乐观 tie-break 再抬高约 38%（0.1802 vs strict 0.1204）。

**写作口径（硬约束）**：主表用 **topic 交叉验证**；`--split-mode document` 只作「论文数字来源解释」，
绝不当协议。这与作者定的「对齐学界、指标难看先问学界怎么构建、不换指标绕过」一致。

## 5. 沿途在自己代码里逮到并修掉的坑（都补了回归测试）

1. **平坦分数假象**：首次冒烟 MRR=1.0000。词表只在 train 建 → 测试全部编码失败 → 返回平坦分数 →
   乐观 tie-break 下 gold 全排第 0 → 满分。`mrr_strict` 戳穿了它。已加 `UnscorableInstance`
   （计最差排名 + 单独报 `n_unscorable`，绝不丢出分母）；词表改为 **transductive**（覆盖 train+test，
   与 SeDGPL 的 `to_add.json` 一致，只有 token 清单跨切分）。
2. **token_span 漏 10% 事件**：MAVEN 触发词粘标点（`died.`）/ 大小写不一（`revolution`）→ 严格空白
   匹配漏 3390/33017。加标点+大小写两级兜底，降到 6 节点/14 实例（所有格），且**编码失败即拒绝**
   而非错位到位置 0。
3. **ESC 不连续 mention**：27 个 mention 跳词（`keep a hold on`）→ 复刻 `doCorrect` 加宽为连续 span。
4. **监控脚本把 ssh 失败当进程死亡**：一次 `kex Connection reset` 让监控误报「运行结束」。
   已改三态判断（ALIVE/GONE/ssh 失败），只有成功 ssh 读到 GONE 才算结束。
